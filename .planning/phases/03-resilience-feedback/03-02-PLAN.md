type RetryConfig struct {

// Retry executes the operation with configured retry logic
func Retry(ctx context.Context, cfg RetryConfig, operation func() error) error

// DefaultRetryConfig creates config from app Config
func DefaultRetryConfig(cfg *config.Config) RetryConfig

```go

```

New flow with retry:
```go
rec := recorder.New(cfg)

// Create retry configuration
retryCfg := retry.DefaultRetryConfig(cfg)
retryCfg.ShouldRetry = func(err error) bool {
    // Check if error is classified and retryable
    // Use errors.IsRetryable or check category
    // Return true for NetworkError, false otherwise
}
retryCfg.OnRetry = func(attempt, max int, delay time.Duration) {
    fmt.Printf("[INFO] Retry %d/%d after %v...\n", attempt, max, delay)
}

// Execute recording with retry
if err := retry.Retry(ctx, retryCfg, func() error {
    // Validate RTSP before each attempt (fresh check)
    if err := validator.ValidateRTSP(cfg.URL, 10*time.Second); err != nil {
        return err
    }
    return rec.Record(cfg.URL)
}); err != nil {
    return fmt.Errorf("[ERROR] Recording failed after %d attempts: %w", cfg.RetryAttempts, err)
}
```

Key changes:
1. Import "rtsp-recorder/internal/retry"
2. Create retry config using existing cfg.RetryAttempts
3. Wrap the recording operation
4. RTSP validation runs inside retry loop (fresh check each time)
5. Return formatted error on final failure

Edge cases to handle:
- Context cancellation during retry delay
- Retry attempts exhausted
- Non-retryable error on first attempt (fail fast)
- Success on retry attempt (report which attempt succeeded)
