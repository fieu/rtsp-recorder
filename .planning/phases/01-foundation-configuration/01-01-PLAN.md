## Implementation Decisions (from CONTEXT.md)

**D-01:** Use subcommands pattern (not single command)  
**D-02:** Phase 1 includes `record`, `validate`, `config` subcommands  
**D-04:** Config file `rtsp-recorder.yml` supports: url, duration, max-file-size, retry-attempts, ffmpeg-path, filename-template  
**D-05:** Conservative defaults: 60 min duration, 1024 MB max size, 3 retries  
**D-06:** Config file is optional — tool works with flags alone  
**D-07/D-08:** Use structured format with level tags: `[ERROR]`, `[WARNING]`, `[INFO]`  
**D-11:** Config precedence: CLI flags > environment variables > config file > defaults  
**D-12:** Environment variables use `RTSP_RECORDER_` prefix

## Research Insights

From STACK.md:
- Cobra v1.10.2 with `cobra-cli init --viper` generates standard structure
- Viper provides automatic precedence: flags > env > config > defaults
- FFmpeg validation uses `exec.LookPath("ffmpeg")` for early failure

From ARCHITECTURE.md:
- Config package isolated from CLI — receives struct, not Viper directly
- FFmpeg validation belongs in `internal/validator/ffmpeg.go`
- Config struct uses `mapstructure` tags for Viper unmarshaling

## Key Patterns

Configuration struct (from ARCHITECTURE.md):
```go
type Config struct {
    URL              string        `mapstructure:"url"`
    Duration         time.Duration `mapstructure:"duration"`
    MaxFileSize      int64         `mapstructure:"max_file_size"`
    RetryAttempts    int           `mapstructure:"retry_attempts"`
    FFmpegPath       string        `mapstructure:"ffmpeg_path"`
    FilenameTemplate string        `mapstructure:"filename_template"`
}
```

Viper setup pattern:
```go
viper.SetConfigName("rtsp-recorder")
viper.SetConfigType("yaml")
viper.AddConfigPath(".")
viper.SetEnvPrefix("RTSP_RECORDER")
viper.AutomaticEnv()
viper.SetDefault("duration", "60m")
viper.SetDefault("max_file_size", 1024)
viper.SetDefault("retry_attempts", 3)
```
